# night-runner

**あなたの定型作業を、夜のあいだに AI が無人で実行するプラットフォーム（macOS）。**

寝ているあいだに Claude Code が登録済みの作業を上から片付け、朝には結果がファイルに残っている。
あなたが触るのは**対話だけ**。ファイル・ターミナル・launchd の設定は AI がすべて裏で代行する。

> Naval "Inspiration is perishable" と Taleb のバーベル戦略の実装 ——
> 「触らない自由」を確保しつつ、夜の固定枠で複利を回す装置。

---

## 導入（Claude Code に貼るだけ）

前提は **macOS** と **Claude Code CLI（`claude`）** だけ。あとは全部 AI（スキル）が裏で代行します。

1. このリポジトリを clone し、`night-runner/` ディレクトリで **Claude Code を開く**。
2. 下のプロンプトをそのまま貼る → セットアップ〜動作確認まで自動で進みます:

```text
night-runner をこの Mac にセットアップして。install スキルで launchd 登録
（claude/node の PATH 自動検出・plist 配備・ロード）まで全部やって、終わったら
test-now で「今すぐ」1回実行して動作確認の結果まで見せて。
私はファイル・ターミナル・plist を一切触らない前提で進めて。
```

3. 夜間に任せたい定型作業ができたら、こう話しかけるだけ（`setup-job` スキルが対話で job 化・登録）:

```text
夜のあいだに毎晩やっておいてほしい定型作業があります。対話で job にして登録して。
作業内容は、後輩に教えるつもりで順番に説明します。
```

## 設計思想（4つの看板）

1. **止まらない**: 夜に走らせたら原則完走させ切る。LLM の「区切りが良い」式の早期停止を、
   プロンプトでなく**構造**（強制ループ）で殺す。
2. **でも致命傷だけは防ぐ**: 破壊的変更・課金・本番への不可逆操作だけは止める。
   ユーザー設定ゼロでもコアがデフォルトで守る。"何で止まり何で止まらないか" の非対称設計。
3. **ユーザーはファイル・ターミナル・plist を触らない**: 設定・登録・テスト・配備は全部 AI が代行。
4. **8割で動かして改善する**: 完璧を最初に目指さない。昼に即時テストで回して現実のズレを直す。

## 動く仕組み（一言）

```
launchd（毎晩）→ core/runner.sh → claude -p で登録 job を上から実行
   → runs/<date>-summary.md に進捗を書く
   → runner.sh が started<planned を見て、未完なら自分を再起動（上限 N）
```

## 使い方（すべて対話で）

night-runner ディレクトリで Claude Code を開き、話しかけるだけ:

- 「インストールして」 → `install` スキルが launchd 登録（PATH 検出・plist 配備まで自動）
- 「今すぐテストして」 → `test-now` スキルが本番と同じ環境で1回即時実行して結果を見せる
- 「この作業を夜間に任せたい」 → `setup-job` スキルが対話で job 化・隔離テスト・夜間登録（ユーザーはファイルを触らない）

## 運用の前提：スリープ防止とノートPCの注意

### 夜間に確実に走らせる（スリープ防止）

launchd の定時実行は、**スリープ中の Mac を起こしません**。夜間にスリープしていると job が
走らない（または起床時に遅れて走る）ので、次のどちらかでスリープを防いでください：

- **電源接続＋蓋を開けたまま**にして、AC 電源時の自動スリープを無効化:
  ```sh
  sudo pmset -c sleep 0
  ```
- もしくは、実行時刻の少し前に Mac を**自動で起こす**:
  ```sh
  # 毎日 23:55 に起床 → 0:00 に job 発火
  sudo pmset repeat wakeorpoweron MTWRFSU 23:55:00
  ```

`sudo` が要るので、これは**あなた自身が実行**してください（Claude Code 上なら `! sudo pmset ...`）。
plist 内の `caffeinate -i` は「実行中の再スリープ」を防ぐだけで、「起こす」のは上記設定が別途必要です。

### ⚠ ノートPCの持ち運び注意

重いタスクを処理させたまま **ノートPCを折りたたんで持ち運ぶと、排熱できず端末が高温になり
危険**な場合があります。job 実行中（特に夜間バッチ稼働中）は **蓋を閉じて移動しない**でください。
持ち運ぶときは、処理が止まっていることを確認してから。

## 使う人 / 貢献する人（目的別の入口）

- **使うだけの人**: このリポジトリを **clone** して `install` するだけ。fork は不要。
  あなたが作る job・実行ログ・生成物・設定（`config.sh`）は **すべてローカルに留まり、
  `.gitignore` で追跡対象外**になっている。つまり **clone 元へ push し返さない限り、
  あなたの実作業が公開されることはない**。
- **改善を貢献したい人だけ fork**: コア（`core/`・`.claude/`）の改善 PR を出したい人向け。
  `core/safety/` と `.claude/settings.json` は CODEOWNERS によりオーナーのレビュー必須。

> **プライバシー設計**: `jobs/<あなたの job>/`・`runs/`（ログ/summary/state/safety.log/生成物）・
> `config/config.sh`・`.env` 等は `.gitignore` 済み。追跡されるのはコード・テンプレ・
> サンプル（`jobs/sample-hello/`）だけ。実データが誤って commit/公開されない既定になっている。

## 構成

| 場所 | 役割 |
|---|---|
| `core/` | 再利用コア（ユーザーは触らない）。runner / plist テンプレ / プロンプト / 安全境界 |
| `config/config.sh` | マシン固有設定（install が生成・`.gitignore` 済み） |
| `jobs/` | あなたの定型作業（self-contained な job 定義。スキルが生成） |
| `.claude/skills/` | 対話の入口（install / test-now / setup-job …） |
| `runs/` | 実行結果（summary + log）。`.gitignore` 済み |
| `docs/` | アーキテクチャと安全設計の正本 |
